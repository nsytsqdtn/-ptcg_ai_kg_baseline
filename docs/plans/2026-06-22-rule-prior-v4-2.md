# RulePrior v4.2 Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add deck knowledge, hard survival/finish gating, and context-aware effect resolution so the pure rule agent follows the v4.2 handoff instead of generic local card scoring.

**Architecture:** The implementation is staged. First add persistent deck knowledge and expose it to state analysis. Next move must-survive and verified-finish logic into inference and main action selection. Then replace generic top-N selection with context-aware choosers for Hilda, Poffin, Ultra Ball, and Petrel. Each stage lands only after failing tests are observed.

**Tech Stack:** Python, pytest, existing `cg.api` observation objects, local battle test harness.
